# 2026-03-16 新方向筛选文档

这份文档只做一件事：把下一轮可做的方向收窄到真正值得做的候选项。

本轮默认淘汰标准：

- 不能只展示 Agent 很忙，必须对真人产生明确价值
- `A2A` 必须是核心机制，不是可有可无的装饰
- 不能只是“社交配对 / 聊天 / 虚拟圆桌”的换皮
- 3 天内必须能做出可演示的 MVP

## 统一评估口径

每个方向都按这 5 个问题判断：

1. 它到底帮真人解决了什么具体问题？
2. 为什么必须靠多个 Agent 协作，而不是单 Agent 或普通工作流？
3. 和别人最容易撞车的点在哪里？
4. 3 天 MVP 最难的部分是什么？
5. 演示时评委能不能一眼看出价值闭环？

## 方向 A：高风险对话预演台

一句话定义：
用户在发出一条重要消息或进入一场高风险对话前，先让多个 Agent 从不同立场做预演，产出更稳的表达方案。

真人价值：
帮助用户处理那些代价高、容易说错的话，比如拒绝合作、谈价格、道歉、催进度、提离职、谈边界。

为什么 `A2A` 是核心：
这里不是让一个 Agent 写文案，而是让多个 Agent 分别扮演“对方视角、风险控制、情绪理解、结果导向”再互相校正。没有 Agent 之间的冲突和修正，这个产品就会退化成普通润色器。

差异化判断：
比“社交匹配”“陪聊”“圆桌讨论”更贴近真实决策场景，价值更直接，也更容易让评委理解“为什么真人会反复用”。

撞题风险：
中等。可能会和“AI 沟通助手”类产品接近，但只要强调多 Agent 互相质疑和风险校正，仍有明显区分。

3 天 MVP 风险：
中等。重点不是大而全，而是把一个高价值场景跑透，例如“给合作方发一条难发的消息”。

## 方向 B：决策前多视角裁判席

一句话定义：
用户带着一个真实选择进入系统，多个 Agent 分别从收益、风险、执行成本、长期后果四个视角辩论，最后给出可执行建议。

真人价值：
帮助用户减少拍脑袋决策，尤其适用于职业选择、项目取舍、合作判断、买不买、做不做、发不发。

为什么 `A2A` 是核心：
如果只有一个 Agent，输出会天然收敛得太快；多 Agent 的价值在于把冲突观点显性化，让用户看见不同判断依据，再由一个汇总 Agent 给出最终建议。

差异化判断：
这个方向比传统“AI 助手答疑”更像一个结构化决策工具，真实使用频率可能很高，且更容易沉淀成演示闭环。

撞题风险：
较低。常见的是单 Agent 建议器，多 Agent 裁判席相对少见。

3 天 MVP 风险：
低到中。最容易落地，也最容易做出前后对比效果，但视觉表现如果太普通，容易显得不够惊艳。

## 方向 C：会后行动生成器

一句话定义：
用户把一次会议、聊天或活动后的信息丢进系统，由多个 Agent 分工提炼“谁该跟进、下一步做什么、现在就该发什么”。

真人价值：
解决会后最常见的问题：记住了很多点，却没有形成明确行动，最后关系和机会都流失。

为什么 `A2A` 是核心：
一个 Agent 负责提炼事实，一个负责识别承诺和待办，一个负责判断优先级，一个负责生成下一步动作。多 Agent 协作的结果是把模糊信息变成可执行清单。

差异化判断：
它直接面向现实里的低执行率问题，比“认识谁”更接近结果导向，也更容易讲“价值不是热闹，而是推进事情”。

撞题风险：
低。和常规会议纪要工具不同，这个方向强调行动决策，而不是摘要。

3 天 MVP 风险：
中等。关键在于输入源要足够简单，建议第一版只支持手动粘贴会议要点或聊天摘录。

## 当前推荐

推荐优先顺序：

1. 方向 A：高风险对话预演台
2. 方向 C：会后行动生成器
3. 方向 B：决策前多视角裁判席

推荐理由：

- 方向 A 最符合“对真人产生实际价值”这个核心要求
- 它不是泛泛的 AI 助手，而是一个高频、刚需、容易让人立刻理解的场景
- `A2A` 在这个方向里不是装饰，而是价值来源本身
- 它和当前常见的 hackathon 展示型社交项目重合度更低

## 不建议再做的旧型方向

以下类型默认谨慎，除非有非常强的新切口：

- 多 Agent 社交配对
- 看起来热闹但没有真实结果的圆桌讨论
- 单纯展示人格、标签、画像
- 只强调“Agent 很智能”，但没有明确用户收益

结论：
下一轮选题优先找“真人马上能用、用了就能避免损失或提高结果”的场景，再把 `A2A` 放进关键决策点，而不是先想一个炫的 Agent 表演。
